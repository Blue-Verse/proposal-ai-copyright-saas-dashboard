핵심 요구사항 분석

  • 턴키 구축 범위: 권한별 화면 기획·UI/UX 디자인, 고객 대시보드·관리자 웹 프론트엔드, 인증/권한/회원/AI 엔진 중계 미들웨어, AWS 인프라 및 CI/CD 파이프라인 전체
  • 멀티테넌시 구조: 여러 기업이 하나의 플랫폼을 공유하되 데이터 완전 격리, 관리자·운영자·조회자 3단계 권한 분리, 감사 로그 상시 기록
  • 콘텐츠 관리: 콘텐츠 업로드·라이브러리, 보호 정책 설정 화면, 각종 정책 설정 페이지
  • 탐지 및 대응: 실시간 침해 탐지 피드, 증거 패키지 조회·다운로드, Notice & Takedown 요청 추적(제출 상태와 후속 조치까지)
  • 알림 및 리포트: 이메일·SMS·인앱 알림, 차트 기반 분석 화면과 월간 리포트
  • 다국어: 한국어·영어 이중언어 지원
  • AI 엔진 연동: 클라이언트 제공 Docker 이미지(저작권 보호 백엔드 엔진)와 안정적 통합, Swagger/OpenAPI 스펙 기반 미들웨어 중계
  • 인프라: AWS 기반(EC2/ECS·RDS·S3·CloudFront 등), CI/CD 파이프라인, SSL/TLS, 도메인 설정 포함
  • 완성도 기준: MVP가 아닌 실제 상용 판매·운영 가능한 안정성과 완성도

제안 포인트

  1. 유사 B2B SaaS 대시보드 다수 수행 — EZ-Approve(전자결재 SaaS, 50+ 페이지, 7 사용자 역할, 6계층 보안), Series-B(VC 펀드 관리, 22 도메인 모듈, 200~300+ API 엔드포인트), Harmony-Link(시니어 케어 멀티테넌트, AI 분석 연동, 140+ 엔드포인트)를 1인 풀스택으로 끝까지 끌고 가본 경험을 그대로 적용합니다.
  2. 멀티테넌시 + RBAC + 감사 로그 패턴 검증 완료 — Harmony-Link에서 멀티테넌트 아키텍처, EZ-Approve에서 CASL 기반 RBAC와 감사 로그를 운영 환경에 안정적으로 안착시킨 경험이 있습니다. 동일한 패턴을 본 프로젝트의 관리자·운영자·조회자 권한 분리 및 기업 데이터 격리에 적용합니다.
  3. AI 엔진(Docker) 연동 미들웨어 설계 강점 — Harmony-Link의 OpenAI 분석 파이프라인 및 AI Agent 프레임워크에서 외부 LLM/추론 엔진 호출 중계·재시도·실패 큐를 다룬 경험이 있어, 클라이언트 제공 Docker 엔진과의 통합을 빠르고 견고하게 처리합니다.
  4. 상용 UI 프레임워크(MUI v5) 기반 효율적 개발 — 우대 사항으로 명시된 MUI v5를 활용해 디자인 공수를 절감하고 컴포넌트 일관성을 확보합니다. EZ-Approve, Harmony-Link 등에서 동일 스택으로 50+ 페이지를 안정적으로 운영해 본 패턴이 그대로 이식됩니다.
  5. 다국어(KR/EN) UI 실전 경험 — Harmony-Link, Fortune App에서 한국어·영어 이중언어 UI를 next-intl/Flutter intl로 구현한 경험이 있어, 본 프로젝트의 KR/EN 이중언어 요구사항에 그대로 대응합니다.
  6. AWS + CI/CD 턴키 인프라 운영 경험 — Harmony-Link(AWS CDK), Series-B(AWS 5계층 보안+DLP+GeoIP), Life3(AWS KMS) 등에서 EC2/ECS·RDS·S3·CloudFront·ACM·Route53·GitHub Actions 기반 CI/CD를 직접 설계·운영해 본 경험을 그대로 적용합니다.
  7. 1순위 산출물 완성도 우선 일정 설계 — 공고에 명시된 "3~4개월" 범위 중 4개월(120일)을 권장 일정으로 제안합니다. 단계별 마일스톤 검수와 통합 테스트 기간을 충분히 확보하여 상용 배포 직후의 안정성을 보장합니다.

PRD (Product Requirements Document)

공고에 명시된 과업 범위와 우대 사항을 반영하여 본 프로젝트의 기능 요구사항을 정리합니다.

1. 프로젝트 배경 및 목적

  • 배경: 클라이언트는 자체 개발 완료한 저작권 보호 AI 백엔드 엔진(API, Docker 이미지)을 보유하고 있으며, 이를 기업 고객이 활용할 수 있는 B2B 플랫폼으로 상용화하고자 합니다.
  • 목적: 고객사가 자사 콘텐츠의 저작권 침해 현황을 실시간으로 확인하고 Notice & Takedown 등 대응 조치를 일관된 워크플로우로 관리할 수 있는 멀티테넌트 B2B SaaS 대시보드를 구축합니다.
  • 완성도 기준: MVP가 아닌 실제 상용화(판매·운영) 가능한 수준의 안정성·UX 완성도를 요구합니다.
  • 턴키 범위: 기획·디자인 → 프론트엔드 → 미들웨어 → AWS 인프라·CI/CD까지 단일 파트너사가 모두 책임지고 납품합니다.

3. 기능 요구사항

3.1 인증 및 권한 관리

  • [필수] 이메일/비밀번호 회원가입, 로그인, 비밀번호 재설정(메일 토큰 기반)
  • [필수] 역할 기반 접근 제어 — 관리자·운영자·조회자 3단계 권한 분리
  • [필수] 멀티테넌시 구조 — 여러 기업이 하나의 플랫폼을 공유하되 데이터·UI 컨텍스트 격리
  • [필수] 감사 로그 — 인증·콘텐츠·정책·Takedown 등 주요 액션 시점·주체·대상 기록 및 조회
  • [선택] 2단계 인증(2FA), SSO(Google Workspace/Microsoft 365)

3.2 콘텐츠 관리

  • [필수] 콘텐츠 업로드(대용량 파일 멀티파트, S3 Pre-signed URL)
  • [필수] 콘텐츠 라이브러리(검색·필터·태그·페이지네이션)
  • [필수] 보호 정책 설정 화면 — 콘텐츠별/카테고리별 보호 수준·우선순위·자동 대응 규칙
  • [필수] 각종 설정 페이지 — 알림 채널, 리포트 수신자, 워크플로우 단계 등

3.3 탐지 및 대응

  • [필수] 실시간 침해 탐지 피드 — AI 엔진이 푸시하는 침해 이벤트를 미들웨어가 수신·정규화하여 대시보드에 스트림(WebSocket)
  • [필수] 증거 패키지 조회 및 다운로드 — URL·캡처·해시 등 메타데이터와 원본 증거 파일
  • [필수] Notice & Takedown 요청 추적 — 요청서 작성·제출, 제출 상태(Pending / Accepted / Rejected / Completed) 및 후속 조치(재요청, 법무 에스컬레이션) 워크플로우 기록
  • [필수] AI 엔진 API 호출 중계 — 미들웨어가 인증·rate limit·재시도·실패 큐를 책임지고 호출을 안정화

3.4 알림 및 리포트

  • [필수] 이메일(AWS SES), SMS(AWS SNS), 인앱 알림 — 채널별 ON/OFF 및 임계치 설정
  • [필수] 차트 기반 분석 화면 — 침해 건수, 처리율, 채널별 분포 등 (Recharts)
  • [필수] 월간 리포트 — PDF 자동 생성·이메일 발송, 다운로드 보관함

3.5 다국어

  • [필수] 한국어·영어 이중언어 UI (next-intl 기반 키 관리)
  • [필수] 사용자별 언어 설정 저장 및 로그인 시 자동 적용
  • [필수] 이메일 템플릿 다국어 분기

3.6 인프라 및 배포

  • [필수] AWS 기반 운영 환경(EC2 또는 ECS Fargate, RDS PostgreSQL, S3, CloudFront)
  • [필수] 클라이언트 제공 Docker 엔진과의 통합 배포(docker-compose 또는 ECS Task Definition)
  • [필수] GitHub Actions 기반 CI/CD 파이프라인, ACM SSL/TLS, Route53 도메인 설정

5. 기술 제약사항

  • 저작권 보호 AI 백엔드 엔진은 클라이언트가 Docker 이미지·OpenAPI 스펙·통합 테스트용 샘플 데이터 형태로 제공합니다. 본 프로젝트는 해당 엔진 자체를 수정하지 않고 미들웨어를 통해 안전하게 중계합니다.
  • AI/ML 모델 및 저작권 보호 알고리즘은 클라이언트 측에서 처리하며, 본 프로젝트의 책임 범위가 아닙니다.
  • 인프라는 AWS를 기본으로 합니다(EC2/ECS·RDS·S3 등 명시). 타 클라우드 이전은 범위 외입니다.
  • 한국어 기본·영어 보조의 이중언어 UI를 기본 정책으로 합니다.
